# 추가 기능 추천

현재 MVP에 추가할 수 있는 기능들을 우선순위와 난이도별로 정리했습니다.

---

## 🎯 우선순위 높음 (핵심 기능 강화)

### 1. **폴더 구조 업로드** ⭐⭐⭐
**난이도**: 중간  
**소요 시간**: 2-3일  
**설명**: 
- 현재는 파일만 업로드 가능
- 디렉토리 전체를 업로드할 수 있도록 개선
- 폴더 드래그 앤 드롭 지원
- 폴더 내부 구조 유지

**구현 방법**:
- File API의 `webkitRelativePath` 활용
- 재귀적으로 폴더 구조 파싱
- GitHub Tree API 사용하여 한 번에 업로드

**사용자 가치**: ⭐⭐⭐⭐⭐ (매우 높음)

---

### 2. **파일 편집 및 삭제** ⭐⭐⭐
**난이도**: 쉬움  
**소요 시간**: 1-2일  
**설명**:
- 레포지토리의 기존 파일 목록 조회
- 파일 내용 편집 (텍스트 파일)
- 파일 삭제 기능
- 파일 이름 변경

**구현 방법**:
- `octokit.repos.getContent()`로 파일 목록 조회
- `octokit.repos.createOrUpdateFileContents()`로 수정
- `octokit.repos.deleteFile()`로 삭제

**사용자 가치**: ⭐⭐⭐⭐ (높음)

---

### 3. **커밋 메시지 커스터마이징** ⭐⭐
**난이도**: 매우 쉬움  
**소요 시간**: 0.5일  
**설명**:
- 업로드 시 커밋 메시지 직접 입력
- 기본 메시지 제공하되 수정 가능
- 커밋 히스토리 확인

**구현 방법**:
- 파일 업로드 전 메시지 입력 모달
- `uploadFiles` 함수에 메시지 파라미터 전달

**사용자 가치**: ⭐⭐⭐ (보통)
